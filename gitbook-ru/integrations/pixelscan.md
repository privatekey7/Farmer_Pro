# Pixelscan

Используется для оценки качества прокси в модуле **Proxy Checker**. Каждый прокси проверяется через запрос к `pixelscan.net`, который анализирует характеристики соединения и возвращает оценку качества.

## Назначение

| Модуль | Роль |
|--------|------|
| Proxy Checker | Определяет `quality` и `latency_ms` для каждого прокси |

## Как работает

```python
result = await check_quality(proxy)
# {"quality": "high" | "medium" | "low" | "unknown", "latency_ms": int}
```

1. Отправляет GET-запрос через прокси на `pixelscan.net` с браузерными заголовками
2. Измеряет время ответа (latency)
3. Парсит поле `quality` из JSON-ответа

## Уровни качества

| Качество | Означает |
|---------|---------|
| `high` | Прокси не детектируется, низкая задержка |
| `medium` | Прокси работает, но частично детектируется |
| `low` | Прокси явно определяется как дата-центр или прокси |
| `unknown` | Ответ получен, но поле `quality` отсутствует |

{% hint style="warning" %}
Pixelscan работает только с HTTP-прокси. SOCKS5-прокси не поддерживаются и вернут ошибку.
{% endhint %}

{% hint style="warning" %}
Если прокси недоступен или возвращает ошибку — выбрасывается исключение. Proxy Checker перехватывает его и записывает статус `ERROR`.
{% endhint %}

## Таймаут

Запрос ограничен **15 секундами**. Прокси с задержкой выше этого значения считаются нерабочими.
